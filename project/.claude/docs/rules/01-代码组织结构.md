## 依赖注入规范

### 核心原则

**禁止使用构造方法注入，优先使用 `@Autowired` 注入。**

### 代码示例

#### ❌ 错误：构造方法注入

```java
@RequiredArgsConstructor
@Slf4j
@Service
public class FoodScaleReportClickHouseServiceImpl implements IFoodScaleReportClickHouseService {
    private final FoodScaleReportMapper foodScaleReportMapper;  // 禁止
}
```

#### ✅ 正确：@Autowired 注入

```java
@Slf4j
@Service
public class FoodScaleReportClickHouseServiceImpl implements IFoodScaleReportClickHouseService {

    @Autowired
    private FoodScaleReportMapper foodScaleReportMapper;  // 正确
}
```

### 适用场景

- 所有 Service 实现类
- 所有 Task 任务类
- 所有 Config 配置类
- 所有需要依赖注入的 Bean

### 注意事项

1. **不使用 `@RequiredArgsConstructor`**：该注解会自动生成构造方法，与本规范冲突
2. **统一使用字段注入**：在字段上使用 `@Autowired` 注解
3. **保持一致性**：项目中所有依赖注入方式应保持一致

## 异步调用规范

### 核心原则

**所有异步调用必须使用 `@Resource(name = "threadPoolTaskExecutorConfgig")` 注入线程池，并使用 `CompletableFuture.runAsync()` 执行。**

### 代码示例

#### ✅ 正确：使用指定线程池执行异步任务

```java
@Resource(name = "threadPoolTaskExecutorConfgig")
ThreadPoolTaskExecutor threadPoolTaskExecutorConfgig;

CompletableFuture.runAsync(() -> {
    try {
        for(CommunityHotTopicCount communityHotTopicCount : finalCommunityHotTopicCountList){
            Long topicId = communityHotTopicCount.getTopicId();
            statisticsInteractionAnalysisService.convertAndInsertCommunityPostForTopicVO(topicId, dates);
        }
    } catch (Exception e) {
        e.printStackTrace();
    }
}, threadPoolTaskExecutorConfgig);
```

### 注意事项

1. **统一线程池**：必须使用 `threadPoolTaskExecutorConfgig` 作为异步任务的线程池
2. **使用 `@Resource`**：按名称注入线程池 Bean
3. **异常处理**：在异步任务内部添加 try-catch 处理异常
4. **禁止使用默认线程池**：不要使用 `CompletableFuture.runAsync()` 不带线程池参数的重载方法
