# 03-设计模式

## 项目中使用的设计模式

### 1. 工厂模式 (Formatter)

Zendesk模块使用工厂模式创建格式化器:

```
BaseFormatter (抽象父类)
    ↓
ModelFormatter, BrandFormatter, CountryFormatter 等 (具体实现)
    ↓
FormatterTypeConstant 获取格式化器类型
```

### 2. 策略模式

根据 `FormatterType` 枚举选择不同的格式化策略:

```java
public static BaseFormatter getFormatter(FormatterType type) {
    return switch (type) {
        case MODEL -> new ModelFormatter();
        case BRAND -> new BrandFormatter();
        case COUNTRY -> new CountryFormatter();
        // ...
    };
}
```

### 3. 单例模式

Spring Bean 默认单例，Service/Mapper均为单例

### 4. 模板方法模式

XXL-Job任务基类定义执行模板，子类实现具体逻辑

### 5. 责任链模式

EasyExcel数据写入使用自定义Handler处理

## 常用设计原则

| 原则 | 应用 |
|------|------|
| 单一职责 | 每个类/方法只做一件事 |
| 开闭原则 | 通过继承/接口扩展而非修改 |
| 依赖注入 | 通过@Autowired或构造函数注入 |
